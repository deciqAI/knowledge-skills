# Method in Action: The Physics of AI Inference Cost (2024–2026)

> *Example for the [first-principles](../SKILL.md) skill.*

A worked example of the First-Principles Teardown applied to a live, contested market belief — the claim, heard in both directions across 2024–2026, that "AI is too expensive to deploy" (or its mirror, "AI is basically free now"). Both slogans copy an incumbent's cost structure by analogy. The teardown decomposes the cost of serving a token down to bedrock — energy, silicon, memory bandwidth, packaging — and rebuilds the strategy conclusion from there.

**The claim (step 1):** "Serving a large language model costs roughly what the leading API providers charge per token — so an AI product's unit economics are fixed by that price." Founders and analysts routinely anchored on published per-token API prices as though they were a floor, exactly the way 2002 aerospace treated the incumbent launch price (see the [SpaceX example](spacex-rocket-cost-2002.md)).

**Surface and tag (steps 2–3):** Decompose "what a token costs" instead of accepting the API price.

- "A generated token requires a fixed number of floating-point operations set by the model's parameter count and architecture" — **BEDROCK**: the compute-per-token of a dense transformer is dominated by its weights, and reads out as arithmetic, not opinion. For a dense model it takes on the order of two FLOPs per parameter per token (Kaplan et al., 2020, scaling-laws compute accounting).
- "Those FLOPs consume energy, and the energy per operation is set by the process node and the accelerator" — **BEDROCK**: joules-per-FLOP is a hardware property, citable per chip generation and improving each node.
- "At batch scale, decode speed is bounded by how fast weights stream from memory, not by raw FLOPs" — **BEDROCK**: autoregressive decoding is memory-bandwidth-bound; this is why accelerators advertise HBM bandwidth, not just FLOPS (widely documented in GPU inference literature and vendor specs, 2023–2025).
- "The token price the API charges is what AI costs" — **INHERITED**: a market price set by a provider's margin, model choice, and strategy — not a production cost.
- "You need the largest frontier model, so you inherit its cost" — **INHERITED**: an assumption about *which* model the task requires, not about the task.

**Interrogate (step 4):** Is the API price generated by physics or by structure? Trace it down and it decomposes into the provider's chosen model size, their hardware utilization, their gross margin, and competitive positioning — business structure, not material necessity. **DEMOLISHED** as a floor on what serving a token must cost. Two forces reported through early 2026 confirm the price was never bedrock: (a) reported per-token prices for a *fixed capability tier* fell sharply year over year as competition and efficiency compounded; and (b) open-weight models (e.g. Meta's Llama family, and efficient mixture-of-experts releases such as DeepSeek's models reported in late 2024 and early 2025) let builders self-host and pay only the underlying compute-plus-energy, collapsing the gap between market price and production cost. The "need the frontier model" assumption is likewise **DEMOLISHED** for most tasks: a smaller or distilled model, activating far fewer parameters per token (the point of mixture-of-experts), meets the task at a fraction of the FLOPs.

**Stop correctly (step 5):** The regress halts at the hardware floor — joules per FLOP, memory bandwidth per second, and the wafer/packaging cost of the accelerator (including HBM stacks and advanced packaging, which were the binding supply constraint through 2024–2025). For a builder *choosing a model and a deployment*, those are citable bedrock. Reducing further ("why does a transistor switch cost that energy?") would be performance, not rigor — it terminates at device physics the builder cannot change.

**Reconstruct (step 6):** Rebuild AI unit economics from surviving bedrock only:
`cost per token ≈ (active FLOPs per token) × (energy + amortized silicon per FLOP) ÷ (utilization), plus memory-bandwidth-bound decode overhead.`
Every term is a lever a builder actually controls: shrink active parameters (smaller or MoE models, distillation, quantization), raise utilization (batching, better serving stacks), ride the falling energy-per-FLOP of each node, and choose self-hosting versus API per the margin you refuse to pay. The frontier API price is one point on this surface, not its floor.

**Diff against convention (step 7):** Convention was wrong about the floor — the API price was a strategy variable, and the true floor sits far below it at the hardware-physics level, which is why the reported cost of a fixed capability kept falling through early 2026. But convention was right that the floor is real and not zero: energy, silicon, and memory bandwidth are hard physical costs, so "AI is basically free" is as wrong as "AI is too expensive." The scarce, expensive input is bedrock — power and advanced-packaging capacity — not the per-token sticker price. The teardown demolishes both slogans and leaves a cost model built on physics.

The mapped steps:

1. Claim stated: "AI unit economics are fixed by the per-token API price."
2. Assumptions surfaced: FLOPs-per-token; energy-per-FLOP; bandwidth-bound decode; the API price; the need for the frontier model.
3. Tagged: compute/energy/bandwidth physics `BEDROCK`; "API price = cost" and "must use frontier model" `INHERITED`.
4. Interrogated: the API price traced to model choice, utilization, and margin — `DEMOLISHED` as a floor; "must use frontier model" `DEMOLISHED` for most tasks.
5. Regress stopped at joules-per-FLOP, memory bandwidth, and silicon/packaging cost — citable hardware bedrock for a builder.
6. Reconstructed from bedrock: a controllable cost model (active parameters × energy/silicon per FLOP ÷ utilization + bandwidth overhead).
7. Diff: the price floor was strategy, not necessity; but the physical floor is real and nonzero — both hype slogans fail.
